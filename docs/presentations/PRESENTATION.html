The Problem

Manual Identity Verification Does Not Scale

  • Ghana's Communications Minister mandated SIM re-registration through the National Communications Authority — millions of mobile subscribers must verify their identity or lose service access
  • The existing process relies on manual agents at physical service centres verifying IDs by eye, creating long queues and operational bottlenecks
  • No automated face comparison means someone can walk into a centre with another person's Ghana Card and register a SIM in their name
  • No audit trail exists — there is no tamper-proof record of who was verified, when, by whom, or with what level of confidence
🕑

Operational bottleneck

Physical counters cannot handle thousands of concurrent verification requests

💰

Cost at scale

Staffing, rent, and infrastructure overhead for every service centre nationwide

⚠

Identity fraud

No face matching means spoofing is undetectable — the exact vulnerability this mandate was designed to close

2
Our Solution

Automated Cloud-Based KYC Verification

The Ghana Card already carries everything needed for verification — photo, name, and a unique PIN issued by the National Identification Authority. We built a serverless pipeline on AWS that takes a Ghana Card photo and a live selfie, extracts identity data via OCR, compares the faces, and returns a PASS/FAIL verdict in under 4 seconds. No agents. No queues. Scales to thousands of concurrent requests.

<3s Verification time
98%+ Face match accuracy
$0.001 Cost per verification
100% Audit trail coverage

How It Works

KYC Verification Flow

1

User uploads Ghana Card and selfie

Images are Base64 encoded in the browser and sent to API Gateway via POST /kyc

2

Lambda stores images in S3 and validates faces

Both images are uploaded to the KYC sessions bucket. DetectFaces confirms a face exists in each

3

DetectText extracts name and ID number from the card

OCR reads all text on the card. The system identifies the surname, forenames, and Personal ID Number using positional and label-based extraction

4

CompareFaces matches the ID photo to the selfie

Rekognition compares the two faces with a 70% similarity threshold. Above 70% is a PASS

5

Result stored in DynamoDB, returned to frontend

PASS or FAIL verdict with face confidence, extracted name, ID number, and session ID. Full audit trail

Infrastructure

Terraform — Infrastructure as Code

All AWS resources are defined in Terraform configuration files. A single terraform apply provisions the complete infrastructure from scratch. Every resource is repeatable, version-controlled, and destroyable.

lambda.tf

Two Lambda functions with Python 3.12, 256MB memory, 60s timeout, and automatic code packaging from source

s3.tf

Four S3 buckets with event notifications, CORS policies, and static website hosting for the frontend

api_gateway.tf

HTTP API with POST /kyc route, Lambda integration, CORS enabled for all origins

dynamodb.tf

KYC results table with PAY_PER_REQUEST billing and session_id as the primary key

iam.tf

Least-privilege execution role with scoped access to Rekognition, S3, DynamoDB, and CloudWatch

cloudwatch.tf

Log groups with 14-day retention and error alarms that trigger when either Lambda throws an exception

Lessons Learned

Challenges and How We Solved Them

ChallengeRoot CauseSolution
Ghana Card name extraction inaccurate Rekognition returns field labels and values as separate unlinked lines with no guaranteed structure Switched from label-based to positional extraction. First name-like line is surname, second is forenames (matching Ghana card layout)
Personal ID Number (GHA-...) not detected The GHA number is printed on the holographic section of the card, making it difficult for Rekognition to read Implemented 5-strategy fallback: word-level GHA scan, line-level scan, label search at multiple confidence thresholds, Document Number as last resort
Unsupported image format crashes Lambda InvalidImageFormatException from Rekognition returned raw AWS error to user Added specific exception catch for InvalidImageFormatException returning a clean user-friendly message
CI/CD pipeline failing with "already exists" errors Terraform state was local, so the pipeline tried to recreate all infrastructure from scratch Separated concerns: infrastructure managed locally via Terraform, pipeline deploys code only

Roadmap

Future Enhancements

Identity Verification

  • Liveness Detection — Rekognition Face Liveness API to close the photo-spoofing vulnerability and prevent identity fraud with printed photos or screens
  • NIA Database Integration — Validate extracted data directly against the Ghana National Identification Authority API for real-time confirmation
  • Open-Source Release — Publish the full pipeline on GitHub so other developers and institutions can build on it

Beyond KYC — Rekognition at Scale

  • Disease Detection — Extend the image analysis pipeline to detect plant and crop diseases from uploaded field photos
  • Agricultural Analysis — Use Rekognition to classify crop health, pest damage, and yield estimates for farming communities
  • Batch Processing — Process entire folders of images uploaded to S3 and generate aggregated summary reports
