WE BUILT A GHANA CARD VERIFICATION SYSTEM ON AWS — HERE'S HOW IT WORKS

────────────────────────────────────────────────────────────────

Last week my team built an image analysis pipeline on AWS Rekognition — detecting
labels, faces, and text from images at scale. This week we asked one question:
what real problem can this actually solve? What we shipped next is a working KYC
verification system tied to Ghana's national identity infrastructure — and the
timing could not be more relevant.

────────────────────────────────────────────────────────────────
WHERE IT STARTED
────────────────────────────────────────────────────────────────

The week before, we built a Rekognition pipeline from scratch — an image analysis
system on AWS that detects labels, faces, and text from photos stored in S3.
Solid foundation. This week we pushed it further and connected it to something
that actually matters.

We brainstormed three directions:

  1. Disease Detection — flagging visible symptoms from patient photos
  2. Crop Disease Detection — identifying plant diseases to support farmers
  3. KYC / SIM Verification — automated identity checks using the Ghana Card

After a team discussion and vote, KYC won. Not because it was the simplest path —
but because the moment demanded it.

The other two remain live extensions of this same pipeline. Disease detection and
crop analysis are directions we're still exploring. But right now, we had a specific
problem worth solving.

────────────────────────────────────────────────────────────────
WHY KYC? WHY NOW?
────────────────────────────────────────────────────────────────

Ghana's Communications Minister recently announced a SIM re-registration exercise
mandated by the NCA. Every mobile subscriber needs to verify their identity or
risk losing access to their line.

That's a massive operational challenge. Millions of people. Physical queues.
Manual checks. Room for fraud at every step.

The Ghana Card — issued by the National Identification Authority — is already
the single source of truth for identity in Ghana. It has a photo, a name, and a
unique PIN. That's all a verification system needs to work.

────────────────────────────────────────────────────────────────
WHAT WE BUILT
────────────────────────────────────────────────────────────────

We extended the pipeline into a full serverless KYC flow. See the architecture diagram above.

At its core: a user uploads their ID card and a live selfie. API Gateway receives
the request and triggers a Lambda function that stores both images in S3, runs
Rekognition to extract the name and ID number from the card, then compares the
card photo against the selfie. Every session is logged to DynamoDB with a
timestamp, confidence score, and PASS or FAIL status. The result comes back to
the frontend in under 4 seconds.

No servers. No manual review. Scales to thousands of concurrent requests.

THE RESULT
────────────────────────────────────────────────────────────────

  Face Match Confidence   99.92%

Verified. Under 4 seconds.

(Demo runs on a dummy Ghana Card and stock photo — no real personal data used.)

Worth noting: the system is not limited to the Ghana Card format. In real-world
use it reads and extracts data from general photo ID cards, and matches the face
on the card against a live selfie taken at the point of verification. The dummy
demo just keeps real identities off the internet.

────────────────────────────────────────────────────────────────
WHAT THIS LOOKS LIKE AT SCALE
────────────────────────────────────────────────────────────────

A system like this gives telcos a real option during the SIM re-registration push:

  - Identity checks automated at scale — no bottlenecks at physical counters
  - Fraud reduced by matching a live selfie to the card on record
  - Every session logged in DynamoDB — full audit trail, tamper-proof
  - Mismatches flagged for human review, not blindly approved

Every component we used is production-grade AWS infrastructure available today.
This is not a concept. We ran it.

────────────────────────────────────────────────────────────────
THE STACK
────────────────────────────────────────────────────────────────

  - Amazon Rekognition (DetectText, CompareFaces, DetectFaces)
  - AWS Lambda (Python 3.12)
  - Amazon S3 (image storage + frontend hosting)
  - Amazon DynamoDB (audit trail)
  - Amazon API Gateway (HTTP API v2)
  - Terraform (full infrastructure as code — reproducible from scratch)

────────────────────────────────────────────────────────────────
THE REAL LESSON
────────────────────────────────────────────────────────────────

The biggest shift wasn't technical. It was in how we framed the work.

A pipeline that detects faces and reads text is a demo. A pipeline that verifies
a Ghanaian citizen's identity before their SIM gets activated is infrastructure.
Same code. Completely different stakes.

The question worth asking on every project: who does this actually help, and
when do they need it?

We're still building. The codebase goes public soon.
