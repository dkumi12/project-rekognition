1 Strengthen Error Handling in Lambda (Rubric: Error Handling 10%)

What it is: Your Lambda functions currently have basic try/catch blocks. The rubric specifically asks for validation of unsupported image formats, oversized files, and retry logic when Rekognition is throttled.

Why it matters: Without this, if someone uploads a PDF, a TIFF, or a 20MB image, the Lambda will crash with an unhandled exception rather than returning a clean error message.

File to update: src/app.py

What to add at the top of lambda_handler:

SUPPORTED_FORMATS = ['.jpg', '.jpeg', '.png']
MAX_SIZE_BYTES = 5 * 1024 * 1024  # 5MB (Rekognition API limit)

# Check file format
key = event['Records'][0]['s3']['object']['key']
ext = os.path.splitext(key)[1].lower()
if ext not in SUPPORTED_FORMATS:
    return {'statusCode': 400, 'body': f'Unsupported format: {ext}. Use JPG or PNG.'}

# Check file size
size = event['Records'][0]['s3']['object']['size']
if size > MAX_SIZE_BYTES:
    return {'statusCode': 400, 'body': f'File too large: {size} bytes. Max is 5MB.'}

For retry logic — wrap your Rekognition calls with a simple retry:

import time

def call_with_retry(fn, retries=3, delay=2):
    for attempt in range(retries):
        try:
            return fn()
        except rekognition.exceptions.ProvisionedThroughputExceededException:
            if attempt < retries - 1:
                time.sleep(delay)
            else:
                raise

2 Add CloudWatch Alarm via Terraform (Rubric: Error Handling 10%)

What it is: A CloudWatch Alarm watches your Lambda error rate and alerts if something is going wrong. The rubric asks for CloudWatch monitoring — your logs already exist, but there are no alarms configured.

Why it matters: Logs tell you what happened after the fact. Alarms tell you something is wrong right now. The rubric calls this out specifically under Error Handling & Logging.

File to update: terraform/ — add a new file cloudwatch.tf

Content to add:

resource "aws_cloudwatch_metric_alarm" "lambda_errors" {
  alarm_name          = "rekognition-lambda-errors"
  comparison_operator = "GreaterThanThreshold"
  evaluation_periods  = 1
  metric_name         = "Errors"
  namespace           = "AWS/Lambda"
  period              = 60
  statistic           = "Sum"
  threshold           = 0
  alarm_description   = "Triggers when Lambda throws an error"

  dimensions = {
    FunctionName = aws_lambda_function.image_analysis.function_name
  }
}

resource "aws_cloudwatch_log_group" "lambda_logs" {
  name              = "/aws/lambda/${aws_lambda_function.image_analysis.function_name}"
  retention_in_days = 14
}

Then run: terraform apply from the terraform/ directory.

5. CI/CD Pipeline Setup (GitHub Actions)

A CI/CD pipeline automates the entire deployment process. Every time you push code to GitHub, it automatically packages the Lambda, runs Terraform, and uploads the frontend to S3. No manual steps, no forgetting to update the API URL.

1 What You Need Before Starting

Before the pipeline can work, three things must be in place:

RequirementWhat It IsWhere to Get It
GitHub Repository Your project must be pushed to a GitHub repo github.com — create a new repo and push your project folder
AWS Access Key ID Programmatic credential to allow GitHub to talk to AWS AWS Console → IAM → Users → Security Credentials → Create Access Key
AWS Secret Access Key The password paired with the Access Key above Generated alongside the Access Key — copy it immediately, shown once

Once you have those, add them to GitHub as secrets:

StepAction
1Go to your GitHub repo
2Click Settings → Secrets and variables → Actions → New repository secret
3Add secret named AWS_ACCESS_KEY_ID with your key value
4Add secret named AWS_SECRET_ACCESS_KEY with your secret value

2 How the Pipeline Works — Plain English

When you push code to the main branch, GitHub automatically runs a sequence of jobs in order:

Step 1 — Checkout → GitHub pulls your latest code onto a fresh Ubuntu machine
Step 2 — AWS Auth → Uses your secrets to authenticate with AWS
Step 3 — Package Lambda → Zips your src/ folder into lambda_function.zip
Step 4 — Terraform Init → Downloads providers and connects to your state
Step 5 — Terraform Apply → Updates all AWS infrastructure automatically
Step 6 — Grab API URL → Reads terraform output and injects it into index.html
Step 7 — Deploy Frontend → Uploads the updated index.html to S3

Every push to main runs all 7 steps automatically. The API URL problem is solved permanently — the pipeline injects it fresh every time.

3 Create the Workflow File

Create this file in your project at exactly this path: .github/workflows/deploy.yml

You will need to create the .github/ and workflows/ folders if they do not exist.

name: Deploy Rekognition Pipeline

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: us-east-1

      - name: Package Lambda function
        run: |
          cd src
          zip -r ../lambda_function.zip .
          cd ..

      - name: Setup Terraform
        uses: hashicorp/setup-terraform@v3
        with:
          terraform_wrapper: false

      - name: Terraform Init
        run: terraform -chdir=terraform init

      - name: Terraform Apply
        run: terraform -chdir=terraform apply -auto-approve

      - name: Get API Gateway URL
        id: api_url
        run: |
          API_URL=$(terraform -chdir=terraform output -raw kyc_api_endpoint)
          echo "api_url=$API_URL" >> $GITHUB_OUTPUT

      - name: Inject API URL into frontend
        run: |
          sed -i "s|const API_URL = '.*'|const API_URL = '${{ steps.api_url.outputs.api_url }}'|" frontend/index.html

      - name: Deploy frontend to S3
        run: |
          aws s3 cp frontend/index.html s3://rekkyc-frontend/index.html \
            --content-type "text/html" \
            --cache-control "no-cache"

      - name: Done
        run: echo "Deployment complete. Frontend live at S3 website endpoint."
